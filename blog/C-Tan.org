* C语言程序设计 - 谭
#+BEGIN_SRC c
#include <stdio.h>

int a, b;
scanf("%d, %d", &a, &b);
#+END_SRC
- =stdio.h= 是头文件， =printf= 在这个头文件中，调用需要先导入
- 源程序执行过程：编辑（.c） -- 编译（.obj） -- 链接 -- 执行
- =#define= 定义全局常量，不能被修改
- 标识符（变量名）只能由字母、数字、下划线3中字符组成，且第一个字符必须是字母或者下划线
- 以 0 开头的数是 8 进制数；以 0x 开头的数是 16 进制数
- int 16位，short int 8位
- unsigned int 无符号整数
- 整型变量 int范围：-32768～32767 溢出 7 + 1 = -8
- 一个整数后面加 u 表示无符号数；-12345u 存储方式是，先转成补码，再以无符号数存储（最高为存储数据）
- 浮点数的写法：e前后必须都有数，且后面的数必须是整数
- float 浮点数精度由尾数的个数来决定，后面的数字是没有意义的
- 单引号、双引号需要加转意字符
- \ddd （\012）三位8进制数 对应ascii码中的字符
- 字符变量可以赋予数字值，8位，ASCII码
- 通过 printf 输出的字符串都是 有符号数 130 => -126
- C 中字符串以 '\0' 字符为结束符号；'\0' 不是可显示的字符，也不是控制字符
- C 中没有专门的数据结构保存字符串，使用字符数组保存字符串，系统在字符串的结尾自动添加 '\0' 字符来结束
- 只有静态变量在编译的时候可以被附初值；普通变量在编译的时候不赋值，在程序运行过程中赋值
- 混合计算时 低精度向高精度转换 (char short) -> int -> unsigned -> long -> double <- float
- 强制类型转换得到的是一个新的类型，原来数的类型不变
- i = 3; j = i++; => j=3, i=4 ||| i=3; j=++i; j=4, i=4
- 高位赋值给低位，直接截断即可
- 标准输入输出函数 getchar(), putchar(), scanf(), printf(), puts(), gets()
- 格式输出：%d %ld %4d %o(unsigned) %f(默认6位小数)  %m.nf(一共输出m列，小数部分n个数，不够左边加空格) %-m.nf(和前面相同，就是不够右边加空格) %e(有效数字默认6位小数) %m.ne
- scanf 的输入可以使用空格、tab、回车分开 %*3d（跳过3个数）
- scanf 中如果使用了除格式控制符号之外的（, /）等符号，需要再输入的时候照着输入才行；如果使用了两个空格，就要输入两个或者两个以上的空格才行
- scanf 读入字符的时候不能有空格存在
- scanf 混合输入的时候需要添加空格
- 一维数组初始化
  + int a[10] = {1,2,...10}
  + int a[10] = {1,2,3} (部分初始化，其余为0)
  + int a[10] = {0} (全部初始化为0)
  + int a[] = {1,2,3,4,5} (可以不初始化数组大小)
- 二维数组初始化
  + int a[2][2] = {{1,2}, {3,4}}
  + int a[2][2] = {1,2,3,4}
  + int a[2][2] = {{1}, {3}} (部分赋值，其余为0)
  + int a[][3] = {1,2,3,4,5,6} (不必初始化第一维)
- char 数组可以使用 int 数组来存储，但会浪费空间
- 字符数组的初始化如果不赋值，其值是不可预料的
- 字符串是以字符数组的形式存储的 （chars 字符数组，str 字符串）
  + char[] a = {"I am happy!"} == char[] a = {'I',' ',...,'\0'}
  + '\0' 是字符串的最后一个字符，char[10] = {"China"} 中前5个字符是China，第6个字符是'\0'，其余字符也是'\0'；所以在输出的时候，只输出"China"，因为遇到'\0'就停止输出了
  + scanf 读入以空格分割，不满的用'\0'填充；scanf("%s", a)
  + strcat 字符拼接函数 将str2拼接到str1上，str1必须足够大
  + strcpy(chars, str) 将str复制到chars，str可以是字符串常量，chars必须是字符数组；不能使用赋值语句直接将字符串常量赋值给字符串
  + strncpy(chars, str, n) 将str的前n个字符复制到chars
  + strcmp(str1, str2) 比较，按ASCII码大小比，小写字母>大写字母；a>b
  + strlen(chars) 获取字符串的长度
  + strlwr(str) 小写 strupr(str) 大写
- 函数
  + 形参（函数内部） 实参（传过来的） C中都是值传递，在内存中都是不同的内存单元
  + 在main函数中声明只需要函数名和参数个数、参数类型，形参不必要
  + 整型函数 int 可以不写返回值，不声明
  + 形参是数组时，指定数组大小无用；传入只传数组的地址
  + 静态局部变量，在程序整个运行期间存在，不释放；编译的时候就初始化了（如果不赋初值，自动赋值0或空），而自动变量不赋初值就上不确定的初值；局部静态变量虽然一直存在，但其他函数不能直接访问
  + 寄存器变量 register int i;
  + 外部变量 加或者不加 static 都是静态存储，在编译时候就赋初值，分配内存
  + 静态外部变量 static var 不能在其他文件中访问
  + 静态函数 static 只能在此文件中访问
  + 默认都是外部函数
- C中变量、函数的作用范围在 定义处到文件结尾；如果想在定义前使用变量，需要使用 extern 关键字
- 可以 #undef G 取消宏定义的范围
- 宏定义 只做字符替换，不分配内存
- 带参数的宏定义 #define S(a, b) a * b 宏名（参数） 字符串
- 宏替换不占用运行时间，只占用编译时间
- 指针
  + 存放地址的变量 &取地址；*间接取值
  + *&a = a; &*pointer_a = &a
  + (*pointer_1)++ = a++; *pointer++
  + C规定数组名代表数组中的首元素的地址 因此 int *p = a <==> int *p; p = &a[0]; <==> int *p = &a[0] <==> int *p = a;
  + int a[10]; a表示数组的首地址，a+i 表示的也是地址，第i个元素的地址
  + 函数指针 void process(int x, int y, int (* func) (int, int))
  + int *p[4] 指针数组； int (* p)[4] 只想一维数组的指针
  + void main(int argc, char * argv[])
  + p -> name <==> (*p).name
